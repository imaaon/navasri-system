## Report 2: Runtime Bug Audit

### Defensive code recommendations

**1. `document.getElementById('xxx').value` without null check — 1013 occurrences**

Pattern ที่อาจเกิด `Cannot read property 'value' of null`:
```js
document.getElementById('some-id').value
```

ส่วนใหญ่ไม่เป็นปัญหา (modal เปิดอยู่ element มีจริง) — แต่ถ้า refactor หรือลบ modal element ในอนาคต อาจพังเงียบ ๆ

**Recommendation (Future R25):** ใช้ helper:
```js
function getVal(id, def = '') {
  const el = document.getElementById(id);
  return el ? el.value : def;
}
```

**2. `JSON.parse` without try-catch — 4 occurrences**

- `js/modules/billing/billing-report.js:317` — `_bsBanks = ... JSON.parse(JSON.stringify(bs.banks))`
- `js/core/db.js:112` — `JSON.parse(ls.value)` — localStorage value (อาจ corrupted)
- `js/modules/billing/billing-clinical.js:551, 602` — `restrictions` array parsing

**Risk:** Low (controlled data sources)
**Recommendation:** Wrap in try/catch + fallback
